# プライバシーポリシー

Codex Remote Androidは、Androidスマホから自宅PC上のCodexへ指示を送り、処理結果をスマホへ返すために必要な情報をFirebaseと自宅PC上のPCブリッジで扱います。

## 取得・保存する情報

アプリとクラウドリレーは、次の情報を保存する場合があります。

- Firebase AuthenticationのUID
- Android端末のFCM token
- セッションID、タイトル、状態、作成時刻、更新時刻
- スマホから送信したコマンド本文
- コマンド状態、開始時刻、完了時刻
- PCブリッジが保存した進捗概要
- Codex処理の最終結果またはエラー
- PCブリッジID、heartbeat、queue確認時刻、手動ヘルスチェック時刻

## 利用目的

保存した情報は、次の目的で利用します。

- スマホとPCブリッジを同じユーザーに紐づけるため。
- セッション一覧とコマンド履歴を表示するため。
- PCブリッジがqueued commandを取得してCodexへ渡すため。
- 処理中の状態、進捗概要、最終結果、失敗理由を表示するため。
- 完了または失敗時にプッシュ通知を送るため。
- セットアップ不備やPCブリッジ停止を診断しやすくするため。

## 通知について

Firebase Cloud Messagingを使って、コマンド完了または失敗時に通知を送信します。

通知payloadには、セッションID、コマンドID、状態などの識別情報を含めます。通知本文には、結果またはエラーの短い要約が含まれる場合があります。通知は端末のロック画面に表示される可能性があるため、秘匿性の高い内容を送信する場合は端末側の通知表示設定を確認してください。

## リポジトリに保存しない情報

次の情報はGitリポジトリに含めない方針です。

- Firebase service account JSON
- Android署名鍵
- `google-services.json` 以外の秘密設定
- `pc-bridge/config.local.json`
- Codex、GitHub、Firebaseなど外部サービスの認証token
- 個人のローカルパスを含む秘密設定

## 第三者サービス

このプロジェクトは、クラウドリレーとしてFirebaseを利用します。Firebaseに保存される情報は、Firebase / Google Cloudの提供する認証、データベース、通知、Functionsの機能を通じて処理されます。

## データの削除

開発・個人利用のMVPでは、データ削除はFirebase Consoleまたは管理者権限を持つ運用者の手動操作で行います。正式公開時には、ユーザー自身がデータ削除を依頼または実行できる手順を整備します。

## 注意事項

スマホから送信するコマンド本文やCodexの結果には、ソースコード、業務情報、個人情報、秘密情報が含まれる可能性があります。必要以上に秘匿性の高い情報を送信しないでください。
