# Skill: Requirements

## Inputs

- User request.
- Existing `docs/requirements.md`.
- Open GitHub Issues.

## Steps

1. Extract functional requirements.
2. Extract assumptions and constraints.
3. Separate MVP from later enhancements.
4. Add measurable acceptance criteria.
5. Create or update GitHub Issues for implementation-sized work.

## Validation

- Every user-stated goal maps to at least one requirement.
- Requirements do not prescribe unsafe direct PC exposure.
- Out-of-scope items are explicit.
