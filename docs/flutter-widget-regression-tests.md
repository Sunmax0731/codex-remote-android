# Flutter widget regression tests

Issue #152 のFlutter側回帰確認は、`app/test/widget_test.dart` のWidgetテストで管理する。

## 実行方法

```powershell
Set-Location app
flutter test
flutter analyze
```

## 現在のカバレッジ

- セッション一覧: 空状態、読み込み状態、読み込み失敗、検索、グループ絞り込み、セッション作成。
- セッション詳細: コマンド未作成、コマンド読み込み中、読み込み失敗、送信、完了カード、失敗カード。
- セッションドロワー: セッション件数、現在選択中のセッション表示。
- PC接続: ステータス表示、ヘルスチェック要求、CLI既定値保存。
- Firebaseセットアップ: 保存済み設定なしの初期画面、QRペイロード、必須ランタイム設定。
- ロケール: 日本語と中国語の代表的な表示。
- テーマ: ダークテーマ時も主要操作が表示されること。

## Golden testの扱い

現時点ではGolden testは導入しない。理由は、Firebaseセットアップやセッション画面がロケール、フォント、OSレンダリング差分の影響を受けやすく、現在のCI/ローカル検証ではスクリーンショット差分よりもWidgetの状態遷移と文言検証の方が安定して回帰を検知できるため。

Golden testを追加する場合は、フォント固定、端末サイズ固定、locale/themeごとの基準画像更新手順を先に決める。
